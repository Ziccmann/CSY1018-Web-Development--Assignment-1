How I Made My First Portfolio Website

Isaac Chukwubuike Samuel-Chukwudi

04/16/2022

For someone who didn’t code at all before entering the university, I will have to give myself an 8/10 with my current development as a coder and its all thanks to the good lecturers of the university of Northampton. Especially Chris Rafferty who teaches us Web Development (CSY1018) and it might seem like I am saying all this because this particular assignment was given to us by Chris himself but it’s true, I still can’t believe I built a website all by myself in this short time. The key things I learnt were how to correctly tag HTML code, where to use classes, how to properly separate CSS codes from HTML codes, how to link CSS and HTML and how to debug. I use the W3C markup validator to debug my HTML and CSS code, I just simply copy and paste the code in W3C markup validator and it traces all the bugs and errors in my code, it also tells me how to correct those errors and where they are located. Right now, I can say that I am well acquainted with HTML and CSS. Though it wasn’t always easy, I had some challenges along the way which is normal.

Below are some of the challenges I went through will building the website with images included;

In the first image, I had some issues committing my work to my GitHub, it almost took me a day to sort that out but eventually I was able to sort it out. In the second image are errors that the validator found in my code, I was able to locate them and fix all of them. In the last two images, I was trying to put the hamburger icon in the mobile responsiveness but it kept appearing when I click on the close bar and it should be the opposite. I checked several lines of my code to find the issue which I did eventually and it worked.
